Layers Rendering Audit — the Overlays, Their Plots, and Their Math

Scope: Documentation/layers/ — 75 C++/Qt files (~1.1 MB), the layer stack vendored from Sonic Visualiser (svgui/layer, Chris Cannam / QMUL). Every miniature plot on this page is drawn live from the audited formulas. Full report: Documentation/Audit/layers_rendering_audit.md.

1 · How a layer gets sound onto the screen

Every overlay subclasses Layer. A layer never owns its view: each paint() call receives a LayerGeometryProvider supplying the time↔pixel mapping (getXForFrame / getFrameForX), frequency↔y mapping, and the zoom level as a frames-per-pixel quantity. Data arrives through an opaque ModelId resolved in a global registry — the model's type decides which layers can display it.

Two disjoint colour systems: single-colour layers pick from ColourDatabase (each colour flagged for dark or light backgrounds); heat-map layers push values through ColourScale → ColourMapper. Painting is time-budgeted by RenderTimer: 0.1 s soft / 0.2 s hard, never bailing before 15% done, pushing to the hard limit once ≥65% complete.

2 · WaveformLayer — the amplitude envelope

min→max strokemean bandclip ≥ 1.0
per-pixel vertical strokes from summary min→max · lighter absmean band · centre axis · butterfly mode splits ch0 up / ch1 down

Data is never raw samples at normal zoom: a peak-cache model serves Range{min, max, absmean} blocks, frame-quantised to block boundaries so a pixel shows the same data at any scroll position (WaveformLayer.cpp:522). Past 1:1 zoom it switches to oversampled interpolation with per-sample dots.

Linear:  y = my − value·gain·m          (m = half band height, my = band centre)
dB:      dB = 20·log10(|v|);  y-offset = (dB + 80)·m / 80,  floor −80 dB
Meter:   voltage_to_fader(v, m, IEC Preview curve)
Auto-normalize: gain = 1 / max(|min|,|max|) over visible range   (WaveformLayer.cpp:552)

3 · SpectrogramLayer — time × frequency heat

synthetic harmonic stack rendered through the audited chain: magnitude → dB log map → Sunset palette · log frequency axis

The layer derives an FFTModel from raw audio: window (default 1024, Hanning), hop from overlap level (none → 93.75%), and oversampling = zero-padding (fftSize = windowSize × 1..8) which interpolates the frequency axis. Two peak-cache mipmaps serve zoomed-out reads.

frequency = bin · sampleRate / fftSize                    (SpectrogramLayer.cpp:1319)
y         = getYForFrequency(f, minF, maxF, {Linear|Mel|Log})
magnitude scaling: raw FFT ÷ (fftSize/2)  →  voltage      (skipped in Phase/Hybrid)
colour options: Linear · Meter · dB (log) · dB² (log ×2: log x² = 2·log x) · Phase

Peak Frequencies mode keeps only bins that are local peaks over threshold and draws each at its instantaneous frequency — the phase-vocoder reassignment from phase advance between hops (estimator lives in FFTModel, outside this snapshot). Normalization: none · column-max→1 · visible-area min/max · hybrid (column shape × log10(colMax+1) so loud moments stay bright).

4 · Colour3DPlotLayer & the shared renderer

The generic heat map for any dense grid (chromagram, MFCC, plugin output). Same colour pipeline, different bin axis:

linear:  y = h − (bin − minBin)·h / (maxBin − minBin)     (Colour3DPlotLayer.cpp:851)
log:     same interpolation in LogRange::map(bin+1) space

Both heat layers delegate to Colour3DPlotRenderer, whose pipeline order is fixed:

get column → scale → normalise → record magnitude extents
           → peak-pick → distribute / interpolate → display gain

Peaks are never averaged away. Many bins per pixel → max-aggregation; fewer bins than pixels → peak-preserving distribute; x-axis upscaling uses a hand-written nearest-neighbour copy because Qt's fast transform can drop single-pixel peaks (Colour3DPlotRenderer.cpp:930).

Scroll cache: ScrollableImageCache keeps one contiguous valid span; scrolling memmoves scanlines sideways and re-renders only the exposed strip, extending the valid region strictly at its edges (drawing right-to-left when extending leftward) — never a hole. A parallel per-column magnitude-range cache drives visible-area normalization and the dB legend.

5 · Value → colour: the 256-step chain

value ×gain → threshold cut (below → pixel 0 = reserved background)
      → scale map {linear · log · meter-fader · ±1 clamp · |abs| · phase}
      → ×multiple → clamp to mapped range → proportion ∈ [0,1]
      → pixel = proportion·255 + 1
      → (+ rotation, wrapped 1..255) → ColourMapper → RGB      (ColourScale.cpp:104-180)

6 · SpectrumLayer / SliceLayer — one moment, all frequencies

magnitude curvedetected peaks
log frequency axis (0.8-shift compromise) · dB energy scale · peak lines at phase-interpolated frequencies

SliceLayer draws one column of any 3-D model at the view centre — styles: Lines, Steps, Blocks, colour-mapped fills; column sampling nearest / mean / peak. SpectrumLayer specialises it with its own FFTModel (window 4096), a piano keyboard + Hz scale, and crosshairs with harmonic ticks at 2f…99f.

log X axis (avoids log 0 at bin 0, compromise shift 0.8):
  x = w · (log10(p+0.8) − log10(pmin+0.8)) / (log10(pmax+0.8) − log10(pmin+0.8))
dB energy:  norm = (dB − threshold)/(−threshold),  dB = 10·log10 (power) | 20·log10 (root-power)
FFT bias curve: each bin × 1/(windowSize/2)  →  voltage      (SliceLayer.cpp:198, 359)

7 · Annotation overlays — events on top of the sound

TimeValueLayer

curve style: cubic-spline through midpoints · 7 styles incl. stems, segmentation bands, discrete curves

Scales: auto-align · linear (+10% margin) · log · ±1; derivative mode plots successive differences. Segmentation bands = ColourMapper.map(value) at alpha 120.

NoteLayer / FlexiNoteLayer

piano-roll rectangles at frequency height · duration = width
pitch p = round(v), cents = 100·(v−p)
f = 440·2^((p−69)/12) ± cents

FlexiNote: 16-px notes; cursor edge (±8 px) picks drag / left / right / split; pitch snaps to the median of the pitch track under the note.

RegionLayer

segmentation bands + I-beam spans · EqualSpaced scale band-centres each distinct value
y = h − ( h·i/n + h/(2n) )
snap when |Δy| < gap/3

TimeInstantLayer & TimeRulerLayer

instants: full-height lines / alternating bands · ruler: 1→5→10→30→60→300…s "nice" ticks, 10 graded minors

Tick x positions are rounded to zoom-block boundaries so labels don't shimmer while scrolling (TimeRulerLayer.cpp:214).

BoxLayer draws an unfilled rectangle from (t0, value) to (t1, value+|level|), both corners through the shared scale. TextLayer anchors labels at a bare proportion y = h − height·h. ImageLayer fits images aspect-preserving; its ImageRegionFinder flood-fills with an adaptive threshold (|colour|/2) and requires ≥2 similar neighbours before expanding — suppressing 1-px antialiasing filaments.

8 · Shared math kernels

CoordinateScale owns every vertical axis (CoordinateScale.cpp:183-236):

point = map(value)      // identity | LogRange::map | Mel(frequency)
y     = h − h·(point − min)/(max − min)     — log scales interpolate linearly in log space

AudioLevel funnels all dB math: 20·log10 for amplitude, 10·log10 for power, and the IEC "Preview" fader curve for every Meter scale — with the fixed tick ladder −40 −30 −20 −15 −10 −5 −3 −2 −1 −0.5 0 dB shared by the scale painter.

An ScaleApplication tag (Normal / Deferring / Personal / None) decides whether a layer's scale may auto-align with other layers sharing its unit — this is how a note layer and a pitch-track curve line up on one axis.

9 · Observations

  1. Peak preservation is a design invariant. Every resolution reduction — waveform summaries, max-aggregation, the hand-rolled upscaler — chooses peak-keeping over averaging, so transients never vanish at any zoom.
  2. Pixel-stability tricks recur. Block-aligned frame quantisation, ruler ticks rounded to zoom blocks, measurement rects stored as frames + fractional heights: nothing shimmers on scroll or resize.
  3. Colour index 0 is reserved as a below-threshold background that bypasses rotation and mapping — thresholding is structural, not cosmetic.
  4. Two math bodies are external to this snapshot: ColumnOp (normalize / peakPick / distribute) and FFTModel (phase-vocoder frequency estimator) live in svcore; call sites are quoted in the full report.
  5. Transferable to Sample Analysis: the gain→threshold→log→palette chain, widen-only display extents, and the scrollable-strip cache map directly onto the Web_Front client-side spectrogram and the Scananalyzer Extractor tab.
