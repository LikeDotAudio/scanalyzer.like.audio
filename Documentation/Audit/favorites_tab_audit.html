The Favorites tab

Ask: press F while listening → the sample is flagged, turns orange, appears in a Favorites tab ("another Examiner, filtered to favorites"), and a favorites.json manifest is written at the library root, parallel to sample_cloud_manifest.json. Companion prose: favorites_tab_audit.md.

One set, owned by App

  
The F key must work on every tab, and the orange must paint four surfaces — Examiner rows,
  the Extractor file list, the footer ★, and the new tab. That is only sane with a single
  Set<string> in App.tsx, keyed on relative path
  (metadata.path || metadata.name) — never the bare name; every drum folder has a
  Kick.wav. App is also the only writer of favorites.json, debounced
  ~500 ms after the last toggle.

  

    
What F flags: footerItem — the sample the footer readout is on, playing or
    just-finished. Gating on "audio currently running" makes a 0.2 s hi-hat unfavoritable.
    During DIG this is the crate-digging loop itself: samples stream by, taps flag them, digging never stops.

  


  
Sibling of the manifest — never inside it

  
The manifest is a rebuildable cache: every re-scan regenerates it, on web and desktop.
  Favorites are user data. Folding them into the manifest deletes them on the next scan.
  Parallel file, parallel lifecycle:

  
<library root>/
├── sample_cloud_manifest.json   ← rebuildable cache — scans overwrite it
├── favorites.json               ← user data — scans never touch it
└── samples + .PEAK sidecars…

  
{
  "favorites_version": 1,
  "generated_unix": 1789000000,
  "favorites": [
    { "path": "Drums/Kicks/Kick 03.wav", "name": "Kick 03.wav", "favorited_unix": 1789000000 }
  ]
}

  
Objects, not bare paths: favorited_unix gives a "recently favorited" sort for
  free, and future fields (rating, colour tag, note) land without a schema break. Full-English
  field names, version-stamped — house rules.


  
The tab: reuse the Examiner, don't fork it

  

    
OptionShapeVerdict

    
A · Fork — copy ExaminerTab → FavoritesTab~950 duplicated lines
      rejected two Examiners to fix every bug in

    
B · Reuse — mount <ExaminerTab> again under #/favorites, rows pre-filtered
      filteredData.filter(it => favorites.has(keyOf(it)))
      recommended DIG, columns, eye, meters inherited free

    
C · Filter chip — "★ only" in the scope bar, no tabone checkbox
      composes with B later; alone it fails the ask

  

  
Because only the active tab mounts (the Suspense block), the transport
  registration and shared column prefs don't collide — the second mount is an Examiner.
  DIG inside Favorites digs favorites only, because DIG walks the rows it was given. The scope
  bar composes: category chips narrow the favorites like every other tab.


  
The two gaps to close

  

    
GapClosure

    
Web folder handles are requested mode:'read' (App.tsx:122, 203); writing needs readwrite
      the first F press is a user activation → lazily requestPermission({mode:'readwrite'});
      denied → in-memory + localStorage mirror + one banner

    
Desktop has no generic root-file write — the Tauri surface is manifest-specific
      two ~10-line commands mirroring the manifest pattern: read_root_text, write_root_text
