2 · What exists today

renderPreview (ExaminerTab.tsx:367-513) is a ~150-line monolith inside a 966-line component. It always draws everything: spectrum fill+trace (Welch-averaged FFT → 360 log bands, y = bottom − clamp((dB+90)/90)·h), stereo min/max waveforms, windowed-RMS loudness (clamp((dB+60)/60), floor −60), stereo correlation phase (corr = ΣLR/√(ΣL²·ΣR²)), the ADSR envelope, beat grid, regions bar, and a hard-coded three-entry legend. The good news: each draw helper is already a pure (ctx, data, geo, colour) function — the SV paint() contract in miniature. The migration is a formalisation, not a rewrite. And the dropdown pattern already exists: the ⚙ Columns menu with its persisted Set (COLS_KEY).

4 · Deployment strategy

  1. Extraction — no visual change. Move the six existing draws into layer files behind the registry; renderPreview becomes a loop. Gate: screenshot parity with today's view, web + Tauri.
  2. Visibility dropdown. LayerStack + Layers menu (SHOW/HIDE), scanalyzer_examiner_layers_v1 persistence, legend generated from visible layers. Gate: toggling any layer leaves the rest pixel-stable; prefs survive reload.
  3. Cheap new layers. PianoScale (pure drawing) and Slices (one FFT column at the playhead, reusing fftRadix2). Gate: slice peaks match the averaged spectrum on a steady tone; A4 key centres at 440 Hz.
  4. Spectrogram engine + heat layers. stft.ts (lazy, cached per decode) + the ported colour chain; waterfall + 3-D ridge, painted chunked across animation frames (SV RenderTimer). Gate: a sine sweep draws a clean diagonal at any canvas width — the max-aggregate, never-average invariant.
  5. Rows mode. Per-layer placement + the stack/rows switch; weighted lanes sharing one time axis and playhead. Gate: playhead aligned across every lane; narrow/mobile falls back to overlay.

Risks: WebKitGTK canvas throughput on big STFTs (chunked painting, frames capped to canvas width) · localStorage schema drift (versioned key) · scope creep into the 966-line ExaminerTab (touch only the preview in these PRs). Everything stays 100% client-side.
